## 🛠️ 准备工作

### 需要准备什么？

1. **一台电脑**（Windows、Mac、Linux都可以）
2. **现代浏览器**（推荐Chrome浏览器）
3. **稳定的网络连接**
4. **你的学号和密码**（用于登录学校选课系统）

### 检查浏览器版本

确保你的浏览器版本不要太旧：
- **Chrome**: 60版本以上
- **Firefox**: 55版本以上
- **Safari**: 10版本以上
- **Edge**: 79版本以上

**如何检查版本？**
1. 打开浏览器
2. 点击右上角的菜单按钮（通常是三个点或三条线）
3. 选择"设置" → "关于Chrome"（或类似选项）
4. 查看版本号

## 📖 详细步骤说明

### 步骤1：登录学校选课系统

1. **打开浏览器**
   - 双击桌面上的浏览器图标（Chrome、Firefox等）

2. **输入网址**
   - 在地址栏输入：`https://xk.webvpn.scuec.edu.cn/xsxk/`
   - 按回车键

3. **登录系统**
   - 输入你的学号
   - 输入你的密码
   - 点击"登录"按钮

4. **确认登录成功**
   - 看到选课系统的主页
   - 确认右上角显示你的姓名或学号

### 步骤2：打开开发者控制台

**什么是开发者控制台？**
这是一个浏览器内置的工具，用来执行代码和查看网页信息。不用担心，我们只是用它来运行抢课助手。

**操作方法：**

1. **方法一：按F12键**
   - 按键盘上的 `F12` 键
   - 浏览器右侧或底部会出现开发者工具

2. **方法二：右键菜单**
   - 在网页任意位置右键点击
   - 选择"检查"或"检查元素"

3. **切换到Console标签**
   - 在开发者工具中找到"Console"标签
   - 点击切换到控制台界面

4. **清空控制台（可选）**
   - 点击控制台左上角的清除按钮🚫
   - 或在控制台输入 `clear()` 然后按回车

**控制台界面截图说明：**
```
┌─────────────────────────────────────┐
│ > 这里可以输入代码                   │
│ >                                   │
│ >                                   │
│ ▼ 显示执行结果和日志信息             │
│   [日志信息会在这里显示]              │
│   [警告和错误也会在这里显示]          │
└─────────────────────────────────────┘
```

### 步骤3：准备抢课助手代码

1. **打开代码文件**
   - 在项目文件夹中找到 `src` 文件夹
   - 打开 `course-registration.js` 文件
   - 可以用记事本或任何文本编辑器打开

2. **复制代码**
   - 按 `Ctrl+A` 全选所有代码
   - 按 `Ctrl+C` 复制

3. **代码内容说明**
   代码看起来像这样（这是示例，实际代码要长很多）：
   ```javascript
   /**
    * 抢课助手核心模块
    * 负责课程注册的核心逻辑和API调用
    */

   import { CONFIG } from './config.js';

   class CourseRegistrationManager {
       // ... 很多代码 ...
   }
   // ... 更多代码 ...
   ```

### 步骤4：运行抢课助手

1. **粘贴代码**
   - 回到浏览器控制台
   - 点击控制台输入区域
   - 按 `Ctrl+V` 粘贴代码

2. **执行代码**
   - 按回车键执行
   - 如果看到错误信息，不要慌张，继续下一步

3. **解决模块导入问题**
   由于浏览器的安全限制，直接复制模块化代码可能会有问题。
   请改用这个单文件版本：

   ```javascript
   // 单文件版本的抢课助手代码
   (function() {
       'use strict';

       // 配置信息
       const CONFIG = {
           BASE_URL: 'https://xk.webvpn.scuec.edu.cn/xsxk',
           POLLING_INTERVAL: 500,
           COURSE_FULL_KEYWORDS: ['课程已满', '已选满']
       };

       // 核心逻辑代码...
       // [这里需要包含完整的抢课逻辑]

   })();
   ```

4. **确认运行成功**
   - 控制台显示类似："用户界面初始化完成"
   - 页面右上角出现抢课助手控制面板

### 步骤5：使用抢课助手

1. **找到控制面板**
   - 在页面右上角找到一个灰色的小窗口
   - 标题显示"自动抢课工具"

2. **输入课程信息**
   - 在"输入课程ID"框中输入要抢的课程编号
   - 可选：在"输入课程名称"框中填入课程名称（便于识别）

3. **添加多门课程**
   - 点击"添加更多课程"按钮
   - 会出现新的输入框
   - 可以添加多门课程同时抢

4. **开始抢课**
   - 确认所有课程都已输入
   - 点击"开始抢课"按钮
   - 按钮会变成灰色，表示正在运行

5. **监控进度**
   - 打开控制台查看实时日志
   - 成功抢到的课程会显示✅标记
   - 可以点击"查看状态"查看进度

### 步骤6：抢课成功后的操作

1. **确认抢课成功**
   - 控制台显示"抢课成功"消息
   - 可以在选课系统中确认已选课程

2. **停止抢课助手**
   - 点击"停止抢课"按钮
   - 关闭浏览器控制台（按F12）

3. **备份选课信息**
   - 截图保存已选课程列表
   - 记录选课时间和课程信息

## ❓ 常见问题解答

### Q1: 按F12没有反应怎么办？
**A1:** 尝试以下方法：
- 尝试按 `Fn + F12`（笔记本电脑）
- 右键点击网页，选择"检查"
- 确保浏览器版本不要太旧
- 尝试使用Chrome浏览器

### Q2: 粘贴代码后出现错误怎么办？
**A2:** 常见错误及解决方法：

**错误："import" statements are not supported**
- **原因**: 浏览器不支持模块导入
- **解决**: 使用单文件版本的代码

**错误："Unexpected token <"**
- **原因**: 复制了HTML内容而不是JavaScript代码
- **解决**: 重新复制代码，确保只复制JavaScript部分

**错误：没有任何反应**
- **原因**: 代码可能没有正确执行
- **解决**: 刷新页面重新开始

### Q3: 找不到课程ID怎么办？
**A3:** 获取课程ID的方法：
1. 在选课系统中找到目标课程
2. 右键点击课程名称
3. 选择"检查元素"
4. 在代码中寻找类似 `jxbid="123456"` 的属性
5. 这个数字就是课程ID

### Q4: 抢课一直失败怎么办？
**A4:** 可能的原因和解决方法：
- **网络问题**: 检查网络连接是否稳定
- **课程已满**: 确认课程确实还有名额
- **系统繁忙**: 等待一段时间再试
- **权限问题**: 确认已正确登录选课系统

### Q5: 控制面板消失了怎么办？
**A5:** 解决方法：
- 刷新页面重新运行代码
- 检查是否被广告拦截器阻止
- 尝试缩小页面寻找面板

## ⚠️ 注意事项

### 安全提醒
1. **仅用于合法目的** - 只能抢选自己有权选择的课程
2. **遵守学校规定** - 不要违反选课系统的使用条款
3. **保护个人信息** - 不要在不安全的环境下使用
4. **适度使用** - 不要给学校服务器造成过大压力

### 最佳实践
1. **提前测试** - 在抢课开始前先测试工具
2. **准备备选** - 准备多门备选课程
3. **及时停止** - 抢到课后立即停止工具
4. **备份信息** - 保存选课成功的证据

### 性能建议
1. **关闭其他标签页** - 减少浏览器负担
2. **使用有线网络** - 比无线网络更稳定
3. **关闭不必要的程序** - 释放电脑资源
4. **定期刷新页面** - 避免会话过期

## 🔧 故障排除

### 网络问题
**症状**: 页面加载很慢或无法访问选课系统
**解决方法**:
1. 检查网络连接
2. 尝试刷新页面
3. 清除浏览器缓存
4. 尝试使用其他浏览器

### 权限问题
**症状**: 无法登录选课系统
**解决方法**:
1. 检查学号密码是否正确
2. 确认选课时间是否已开始
3. 联系学校IT部门确认账户状态

### 代码问题
**症状**: 控制台显示错误信息
**解决方法**:
1. 仔细检查代码是否复制完整
2. 刷新页面重新开始
3. 尝试使用其他浏览器
4. 寻求技术支持
